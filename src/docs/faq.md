# 常见问题

## 抓不到资源如何做

先尝试[深度搜索](cache-capture#deep-search)。再尝试[缓存捕捉](cache-capture#cache-capture)，该功能需要等待视频缓冲完毕后才能下载。仍然无法下载，可以尝试最后的[屏幕捕捉](cache-capture#screen-capture)。

如果你遇到只有`缓存捕捉`才能下载资源的网站，请到github提交bug 或 发送邮件。

## 音频和视频分开的

很多网站开始使用音频和视频分离的播放技术，只需要勾选一个音频和一个视频，如果符合条件，会出现 `在线合并`按钮，点击进行合并下载。

::: info
技术限制，暂时只能合并小于2G的视频，否则只能使用本地工具合并。
:::

## 使用缓存捕捉 ffmpeg合并出错

出现 `error reading header` 错误，捕获的文件头部信息缺失，取消ffmpeg合并勾选 下载音频和视频 用第三方工具合并。

## 抓不到YouTube

寄人篱下 没办法啊

## 某鹅通无法下载

尝试使用油猴脚本 [https://greasyfork.org/zh-CN/scripts/461963](https://greasyfork.org/zh-CN/scripts/461963)

脚本会尝试获取m3u8重新拼接正确地址 发送给猫抓。

## m3u8资源合并下载后有声音，但视频花屏

该资源使用了非常规的加密方式，目前暂无办法解决，可以尝试[缓存捕捉](cache-capture#cache-capture)。

## m3u8资源合并下载，打开只显示一张图片 / 碎片资源为png格式

该资源把视频碎片伪装成图片，需要在m3u8解析器勾选 `数据预处理` 该功能会剔除不需要的图片数据或使用第三方工具 N_m3u8DL-RE 下载。

## m3u8解析器合并下载中页面崩溃

可能下载的文件太大，你的内存不够，需勾选 `边下边存` ，勾选该功能后无法使用在线转码功能。

## 在线ffmpeg卡在 **ffmpeg加载中...**

在网页命令框 输入 `clearFFmpeg` 命令，回车，清理ffmpeg，然后刷新页面，重新下载...

## 在线ffmpeg加载wasm好慢啊

试试秘籍 `CatFoodForPrawnGuns` 😘

## 下载未完成/失败的文件保存在哪里?

没启用`边下边存`功能的情况下，下载的过程数据会保存在内存中，如果下载失败或未完成，刷新页面/关闭浏览器 数据就会被浏览器自动清理。仍然无法清理可以在浏览器设置中尝试清理缓存。

## 自定义名称失效

不管在 `替换标签` 设置 `自定义保存文件名` 还是m3u8解析器自定义名称，下载文件名都不是设置的名称。可能是你使用了第三方下载工具，或者安装的其他下载器扩展，导致下载API使用冲突。请关闭其他下载器扩展。
