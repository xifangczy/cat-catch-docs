# 设置

## 扩展和类型匹配规则

浏览器接收到文件时进行判断

![抓取后缀](/assets/extension.png)

![抓取类型](/assets/type.png)

抓取类型，也称为[MIME](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Guides/MIME_types)扩展类型，部分网站不提供正确的文件后缀，需使用MIME扩展类型判断文件，子类型支持通配符 `*` 。

允许使用关系运算符，支持 `> >= < <= = !=` 以及范围表达。容量单位支持 `B KB MB GB`\
例如:

`>= 200KB` 大于等于200KB的资源

`< 1GB` 小于1GB的资源

`!=500KB` 不等于500KB的资源

`500-1024 MB` 500MB到1G范围的资源

## 正则匹配

如果一些资源没有后缀，没有MIME类型，可以通过正则进行URL匹配，或想通过正则匹配进行资源筛选屏蔽。

![正则匹配](/assets/regex.png)

**后缀** 正则匹配的资源无法知道资源类型和后缀，需要指定一个后置扩展名，留空会尝试截取网址的后缀名，由于许多网址没有后缀所以可能最终无法知晓类型，无法在popup页面显示相应的解析器入口或播放图标。

**屏蔽资源** 可以屏蔽某些不想抓取的资源。

::: info
正则匹配效率低下，尽量减少正则的使用，避免资源占用过多。
:::

## 屏蔽网址

在部分网址上不需要扩展，可以设置网址，支持通配符，例如 `https://x.com/*`

开启 `设置为白名单` 选项，列表更改为白名单，只有该名单内的网址扩展才会正常工作。

## 其他设置

![其他设置](/assets/other-settings.png)

### 始终不启用猫抓下载器

猫抓内置简易下载器，能够携带原始请求头下载文件，检测到浏览器自带下载器下载失败时会自动调用猫抓下载器重新下载文件，确保下载成功。如果你使用第三方下载工具可能会造成冲突，可以开启该选项禁止猫抓下载器被调用。

### 使用本地播放器调用协议打开视频预览 {#player-protocol}

部分本地播放器支持调用协议，例如PotPlayer，在`调用协议模板`选择PotPlayer，popup页面点击播放按钮会直接调用本地播放器预览。手机用户建议使用 `系统分享` 模式。

### 刷新、跳转到新页面 清空当前标签抓取的数据

选择抓取的资源清空模式:

- 不清空，任何情况都不清理数据，直到用户关闭当前页面的标签。
- 正常清理，用户主动刷新页面或点击跳转到新页面，当前抓取的数据会被清空。
- 更频繁，在正常清理的基础上，会更加敏感，例如切换新视频也会触发清空。

部分网站会自行多次刷新页面，导致抓取数据被清空，遇到嗅探不到资源的网站，请先尝试清理模式改为 不清空。

### 嵌套在线ffmpeg

使用[在线ffmpeg](online-ffmpeg)过程中，因各种复杂原因导致数据传输失败，在线ffmpeg没收到数据，可以打开这个选项，使用嵌套的在线ffmpeg解决。

## 发送数据 {#send}

popup页面点击发送按钮，能够自定义发送数据到第三方地址上，请求体 确保为JSON格式

`${action}`为特殊标签, 表示不同数据类型, 如果是捕获数据替换为 `catch` 字符串, 如果是密钥数据 会被替换为 `addKey` 字符串。

如果`${action}` 是`catch` ${data} 则为 文件对象转为JSON格式内容

如果`${action}` 是`addKey` ${data} 则为 密钥base64字符串

示例: `{"action": "${action}", "data": ${data}, "tabId": "${tabId}"}`

```json
// {"action": "${action}", "data": ${data}, "tabId": "${tabId}"}
{
    action: "catch",
    data: {
        name: "文件名",
        url: "资源地址",
        size: "资源大小"
        requestHeaders: {
            referer: "也可能不存在",
            origin: "也可能不存在",
            cookie: "也可能不存在"
        }
        ...: "其他"
    },
    tabId: "数据来源的tab ID"
}

// JSON 接收到疑似密钥
{
    action: "addKey",
    data: "base64数据",
    tabId: "数据来源的tab ID"
}
```
